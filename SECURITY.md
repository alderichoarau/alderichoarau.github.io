# Security Policy

This repository hosts a static personal portfolio site with no backend, no
user data, and no authentication. The attack surface is limited, but reports
about the following are still welcome:

- A vulnerable dependency (see also `npm audit` / Dependabot alerts).
- An issue in the CI/CD pipeline (GitHub Actions workflows) that could allow
  unauthorized code execution or a supply-chain compromise.
- Any other security weakness in the build or deployment process.

## Reporting a Vulnerability

Please report security issues privately by emailing
[alderic.hoarau@gmail.com](mailto:alderic.hoarau@gmail.com) rather than
opening a public issue. Include as much detail as possible (steps to
reproduce, affected files/workflows, potential impact).

You should expect an acknowledgment within a few days. There is no bug
bounty program; this is a personal, non-commercial repository.

## Scope

As stated in [LICENSE](./LICENSE), this repository is not open source and
external contributions are not accepted (see
[CONTRIBUTING.md](./CONTRIBUTING.md)). Security reports are the only
exception to this policy.
